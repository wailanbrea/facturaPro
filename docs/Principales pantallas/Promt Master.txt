Actúa como arquitecto senior full stack, desarrollador Laravel, Android Kotlin Jetpack Compose, especialista en sistemas de facturación, generación de PDF profesional y diseño de aplicaciones escalables.

Vamos a desarrollar un sistema web y móvil para generar facturas usando como base una plantilla Excel oficial llamada:

Factura P L - Oficial.xlsx

Antes de programar cualquier cosa, debes analizar la plantilla de factura y entender todos sus campos, distribución, fórmulas, secciones, textos legales y comportamiento esperado.

La aplicación debe permitir crear facturas desde web y desde móvil Android. Las facturas deben poder verse con el mismo formato visual de la plantilla original y deben poder guardarse como PDF manteniendo el mismo diseño.

No debes desarrollar todo de golpe. Debes trabajar paso a paso siguiendo un archivo guía llamado TODO.md.

Tu primera tarea será crear el archivo TODO.md con todas las fases del proyecto, checklist técnico, orden de desarrollo, estado de cada módulo y criterios de aceptación. Luego deberás seguir ese TODO estrictamente.

No debes saltar fases.  
No debes asumir reglas críticas sin dejarlas documentadas.

Cálculos obligatorios:

base_linea = cantidad * costo_unitario

Si los precios NO incluyen impuesto:
    impuesto_linea = base_linea * porcentaje_impuesto
    precio_unitario = costo_unitario + (costo_unitario * porcentaje_impuesto)
    importe = base_linea + impuesto_linea

Si los precios SÍ incluyen impuesto:
    base_linea = importe / (1 + porcentaje_impuesto)
    impuesto_linea = importe - base_linea

Totales:
subtotal = suma(base_linea)
impuesto_total = suma(impuesto_linea)
total = subtotal + impuesto_total
balance_pendiente = total - importe_recibido

Regla crítica:
El backend siempre debe recalcular subtotal, impuesto_total, total, balance_pendiente y estado. No confiar nunca en los totales enviados desde web o Android.

Estados de factura:

- BORRADOR
- EMITIDA
- PAGADA

- ANULADA

Reglas:
- Si importe_recibido = 0 y factura está emitida → EMITIDA
- Si importe_recibido > 0 y < total → PARCIALMENTE PAGADA
- Si importe_recibido >= total → PAGADA
- Si fecha_vencimiento < hoy y balance_pendiente > 0 → VENCIDA
- Si se anula manualmente → ANULADA

Reglas de edición:
- BORRADOR: editable completo
- EMITIDA: editable solo con permiso especial
- PAGADA: no modificar montos, solo notas internas
- ANULADA: no modificar
